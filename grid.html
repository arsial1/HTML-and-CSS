CSS Grid is right now the most modern way of building layouts and it's also the most complete one and in my opinion is even the easiest way of building layouts at least if you use only the fundamentals. When CSS Grid was first developed and brought into browsers a couple of years ago, the main idea behind it was that now we could divide a container element into rows and columns that we could then fill with its child elements.

In this post, let's focus on CSS-Grid. We will learn what CSS-Grid is all about, and why you too should learn and use it.

What is CSS Grid?

At the most basic level CSS Grid is a set of CSS properties that web developers can use to build two dimensional layouts.

CSS Grid allows you to create a grid-based layout where you can place website elements in precise locations and easily make responsive designs. You can define columns and rows in your grid, place items in specific grid areas, control the size of the gaps between elements, and much more.

You can explore the CSS-Grid guide at CSS-Tricks to learn more.

How to use CSS Grid

One common misconception that many beginners have is that CSS Grid is meant to replace flexbox. However, that is really not the case. So instead flexbox and CSS Grid work together perfectly. You simply use them in different situations.

So when you need a one dimensional layout, then you can simply reach for flexbox. But if you need a two dimensional layout, which is usually a grid anyway, then you can use CSS Grid for that.

We create a grid container by setting its display property to grid. Then all the child elements of the grid container will be the grid items. 

          

            .container {
                display: grid;
            }

Now, unlike flexbox we cannot change the direction of the row and the column axis, which actually makes it even a bit easier to work with CSS Grid.

However, since two dimensional layouts are a bit more complex, we need actually some more terminology in order to really understand what's going on. So let's look at some more terminology here.

Overview of all CSS Grid properties

So for the CSS Grid container, the most fundamental ones are the grid-template-columns, the grid-template-rows, the column-gap and the row-gap properties. 

          

            .container {
                display: grid;
                grid-template-columns: 200px 1fr 100px;
                grid-template-rows: 300px 200px 2fr 50px;
                column-gap: 20px;
                row-gap: 10px;
            }

Grid Lines: divide up the grid and separate the columns and the rows. They can be vertical (column lines) or horizontal (row lines).

These grid lines are numbered from one to the number of columns plus one, and the number of rows plus one. So if we have three columns, then we have four grid lines for the columns. And if we have two rows, then we have three grid lines for the rows.

These numbers are actually important because we can use them to place a grid item exactly in one place in the grid where we want it to be.

Grid Track: The space between two adjacent grid lines. You can think of them as the columns or rows of the grid.

Grid Cell: The space between two adjacent row and two adjacent column grid lines. It's the single unit of the grid. Grid cells are always created, but they don't always need to be filled.

Grid Area: Any rectangular area enclosed by four grid lines. A grid area can be composed of one or more grid cells.

Fr Unit: A flexible length unit that distributes available space. It allows for the creation of flexible layouts that adapt to the size of the container.

There are also two less important properties on the grid container, which are justify content and align content. These only apply if the container is actually larger than the grid.

Why should you learn CSS Grid?

There are countless and compelling reasons for learning and using CSS Grid:

  • Simplicity:CSS Grid simplifies the process of creating complex layouts. With just a few lines of CSS, you can define a grid and place items precisely where you want them. This simplicity leads to cleaner and more maintainable code compared to older layout methods like floats and positioning.
  • Powerful Layout Control: CSS Grid provides precise control over layout structure, allowing you to create sophisticated designs without relying on nested HTML structures or complex CSS rules. You can easily create multi-column and multi-row layouts, control the size and alignment of grid items, and create responsive designs with minimal effort.
  • Responsive Design: CSS Grid makes it easy to create responsive layouts that adapt to different screen sizes and devices. By defining flexible grid tracks and using features like media queries, you can create designs that look great on everything from large desktop screens to mobile devices.
  • Grid Gap: CSS Grid introduces the grid-gap property, which allows you to specify the spacing between grid items. This feature makes it easy to create visually appealing layouts with consistent spacing between elements.
  • Future-Proofing: CSS Grid is a modern layout solution that is widely supported by modern web browsers. By using CSS Grid, you're adopting a forward-looking approach to web development that will continue to be supported and improved upon in the future.

CSS Grid Summary

  • In two-dimensional contexts, CSS Grid allows us to write less nested HTML and easier-to-read CSS.
  • CSS grid tracks, grid gap and other properties allow you to create visually appealing layouts with consistent spacing between elements.
  • CSS grid is not meant to replace flexbox! Instead, they work perfectly together.
  • Need a 1D layout? Use flexbox. Need a 2D layout? Use CSS Grid.
